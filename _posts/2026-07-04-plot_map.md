As usual first thing first. We need to import the libraries needed in this project. There are a couple of libraries that
I was not familiar with before starting this project. I need to give credit to my buddy [Tom Chudley](https://trchudley.github.io/),

```python
import numpy as np
import geopandas as gpd
import rioxarray as rxr
import matplotlib.pyplot as plt
from matplotlib.colors import LightSource, LinearSegmentedColormap
from rasterio.enums import Resampling

import gc
```

Once we have imported the necessary libraries, we need to load the shape file that we want to use
to clip the DEM file that we want to plot. I downloaded the Appalachian Mountain
boundary shapefile fromt the USGS website for this purpose. I had to do some trimming, but it was pretty
straightforward thanks to the geospatial python library ecosystem that makes 
this process pretty smooth. Once the shape file is trimmed, we just need to load
the shape file using `geopandas`.

Here is the script to load the shape file. After loading the 
shape file, I print out the coordinate reference system or `crs`
so that I can make sure when I perform operations on multiple obejcts they are all in the same `crs.

```python
mtn = gpd.read_file("../shp/appalachian_basin.gpkg")
print("Mask CRS:", mtn.crs)
```

    Mask CRS: EPSG:5070


The next step is loadng the raw DEM file I downloaded from the OpenTopography website. It's good resource to access topography based dataset.

The method `.squeeze()` drops the single-band dimension so we get a 2D array
instead of `(1, H, W)`, whereas `masked=True` turns the nodata value into NaN,
which matters later for hillshading and colormapping.


```python
dem = rxr.open_rasterio("../raster/appalachia_srtm.tif", masked=True).squeeze()
print("DEM CRS:", dem.rio.crs)         # SRTM comes in EPSG:4326
print("DEM shape:", dem.shape)          # sanity check size
print("Elev range:", float(dem.min()), float(dem.max()))
```

    DEM CRS: EPSG:4326
    DEM shape: (17319, 25071)
    Elev range: -171.0 2025.0


The next step is clipping the DEM to the polygon using the `rxr` method `.clip`.

The OpenTopography download is a rectangular bbox — it includes
stuff outside the mountains (coastal plain, etc.). Clipping to the
polygon masks those pixels to NaN so they don't render.

We also need to reproject the mask to match the DEM. If you don't do this, rioxarray
throws a CRS mismatch. Always do this before .rio.clip().


```python
mtn_dem_crs = mtn.to_crs(dem.rio.crs)
dem_clipped = dem.rio.clip(mtn_dem_crs.geometry, mtn_dem_crs.crs, drop=True)

# Free the original — we don't need it anymore.
del dem
gc.collect()
```




    145



One thing to note, the paramater `drop=True` actually trims the raster's extent to the mask's bounding box.
Without it, you keep the full rectangle with NaNs outside the polygon,
which wastes memory. With it, the array itself shrinks.

Before we move on I want to highglight this step
```python
del dem
gc.collect()
```
The syntax `del` delete an object and `gc.collect` perform [**garbage collection**](https://en.wikipedia.org/wiki/Garbage_collection_(computer_science)). Garbage collection is is a form of automatic memory management. This process attempts to reclaim memory that was allocated by the program.

I had to do this in this project (first time I ever use this in a python project), because I notice the runtime of my python getting really long (takes longer than I expected). However when I check the memory it only utilizes a fraction of what is available but my laptop had to use up all the [swap memory](https://www.geeksforgeeks.org/operating-systems/swapping-in-operating-system/). This may have something to do with modern memory management that I am not gonna pretend to understand. Long story short, because of this weird memory overflow, I have to frequently deleting object I no longer need and perform garbage collection. It improves the runtime from 5 minutes to 30 something second.

The next step is reprojecting the clipped dem to an equal-area projection.
This is the single most important step for making the map "look right."
SRTM (the dem I downloaded from OpenTopo) is in EPSG:4326 (geographic lat/lon). If you render as-is, one
degree of latitude looks the same as one degree of longitude on the
page — but at 40°N, a degree of longitude is only 77% as wide as a
degree of latitude in reality. Your map ends up looking horizontally
stretched.

EPSG:5070 (NAD83 / Conus Albers) is the standard equal-area projection
for CONUS maps. It preserves area, which is what you want for a
regional map like this. Distances and angles are slightly distorted
but not enough to notice at the scale I want to print the poster.

I have to downsample the data to make it small enough, and perform resampling. Bilinear resampling is right for continuous data like elevation.
Never use nearest neighbor for DEM, because it creates staircase artifacts (learned it the hardway).


```python
TARGET_RES = 270  # meters

dem_albers = dem_clipped.rio.reproject(
    "EPSG:5070",
    resolution=TARGET_RES,
    resampling=Resampling.bilinear,
)

del dem_clipped
gc.collect()

print("Downsampled shape:", dem_albers.shape)
print("Array size:", dem_albers.nbytes / 1e6, "MB")  # should be well under 1 GB
```

    Downsampled shape: (7089, 7545)
    Array size: 213.94602 MB


This next step is also important. We need to extract the array and handle NaNs.
Matplotlib's LightSource can't handle NaN inputs. It'll propagate
them through the entire array via the gradient calculation and you'll
get a black map. So we replace NaN with 0 (sea level) but keep a
separate mask so we can hide those pixels in the final image.


```python
elev = dem_albers.values.astype(np.float32)
mask = np.isnan(elev)
elev_filled = np.where(mask, 0, elev).astype(np.float32)

print("Final array:", elev_filled.shape, elev_filled.dtype,
      f"{elev_filled.nbytes / 1e6:.1f} MB")
```

    Final array: (7089, 7545) float32 213.9 MB


For aesthetic purpose, I created a custom colormap.

```python
cmap = LinearSegmentedColormap.from_list("appalachia", [
    "#2d4a3e",   # dark forest green (low elevation)
    "#5a7a4f",   # mid green
    "#a89968",   # tan
    "#d4c896",   # pale gold
    "#f0e6d2",   # cream (ridge tops)
])
```

This step, for whatever reasons, is the most computationally heavy step. The hillshade + color blending in LightSource, shown in the snippet below, does two things: computes a shaded relief from the DEM
gradients, then blends it with your colormap.

The parameter `azdeg=315` set light from the NW. This is a convention. Do not change unless you
want a jarring map. Human perception reads NW-lit terrain as convex,
SE-lit as concave (inverted).

The parameter `altdeg=15` set the sun 15° above horizon. The standard is 45, but I choose lower to give
longer shadows and more dramatic relief.

The parameter `vert_exag` set the exaggeration factor. The Appalachians max out around
2000m, spread over ~2000km. Real vertical:horizontal ratio is tiny.
Without exaggeration, the map looks flat.

The parameter `blend_mode="soft"` (Adobe soft-light equivalent) preserves color
fidelity best.


```python
ls = LightSource(azdeg=315, altdeg=15)
rgb = ls.shade(
    elev_filled,
    cmap=cmap,
    vert_exag=25,
    blend_mode="soft",
)
```

The step below make the object outside of the set boundary to be transparent.

```python
rgb[mask, 3] = 0

# Free the intermediates we don't need for saving.
del elev, elev_filled
gc.collect()
```

This last part is plotting and saving the file as a png. This step is pretty self explanatory.



```python
# figsize tuned to the array's aspect ratio so pixels aren't squished.
h, w = rgb.shape[:2]
aspect = w / h
fig_h = 32
fig_w = fig_h * aspect

fig, ax = plt.subplots(figsize=(fig_w, fig_h), dpi=300)
ax.imshow(rgb)
ax.set_aspect("equal")  

ax.set_axis_off()

plt.savefig(
    "../figures/appalachia_hillshade_vert25_altdeg15.png",
    dpi=300,
    bbox_inches="tight",
    pad_inches=0,
    transparent=True,
)

# Final cleanup
del rgb, mask
gc.collect()
```
